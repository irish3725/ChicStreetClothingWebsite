Project Technical Grading (50 points)

Note: The project is worth 100 point. The other 50 points can be found under Project Design Grading.

I. Files Organization. (10 points)

  • The main page must be named index.html 
  • Two other html pages in addition to index.html
  • Only lowercase or camel case letters are used in file names 
  • Do not use spaces in file names 
  • Only necessary files are submitted 
  • Files have good, meaningful names
  • The files are organized logically.
  • Stylesheets are in css folder.
  • Images in images folder.
  • index.html is in main project folder 

II. All three web pages validate as HTML 5 (10 points).

  • 1 type of error- lose 2 points
  • 2 types of error - lose 3 points
  • etc.

Note: 1 additional point will be lost if there are any warnings besides the HTML 5 warning.

III. The html files (10 points)

  • Files are easy to read and indented logically when the source is viewed.
  • The html files are sophisticated.
  • No style appears in the html files.
  • The html files have meaningful titles.
  • Images have meaningful alt descriptions.
  • Navigation is done with an unordered list.

IV. All four css pages validate (10 points).

  • 1 type of error- lose 2 points
  • 2 types of error - lose 3 points
  • etc.

V. The css files (10 points)

  • One global.css file (that contains style common to each page)
  • Three page-specific css files (one for each html page) are used correctly. 
  • Contents are organized logically
  • Contents are indented logically
  • css files are sophisticated
